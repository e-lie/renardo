# Tutorials

Renardo includes a comprehensive set of tutorial files to help you learn how to use the system effectively. These tutorials are interactive and can be executed directly in the editor.

## Accessing Tutorials

The tutorials are accessible from the right panel of the editor:

1. Ensure the right panel is open (click the panel toggle button if needed)
2. Select the "Tutorial" tab in the right panel
3. Browse the available tutorial files

## Tutorial Languages

Tutorials are available in multiple languages:

- English
- Spanish
- More languages may be added in future versions

To change the tutorial language:

1. Open the tutorials panel
2. Use the language selector dropdown at the top
3. Select your preferred language
4. The tutorial list will update to show files in the selected language

## Tutorial Structure

The tutorials are organized in a progressive sequence, starting with basic concepts and moving toward more advanced features:

1. **Introduction** - Basic overview of Renardo
2. **Playing Notes** - Creating your first sounds
3. **Algorithmic Manipulation** - Modifying patterns programmatically
4. **Playing Samples** - Working with audio samples
5. **Using Patterns** - Advanced pattern creation
6. **Player Attributes** - Customizing sound parameters
7. And many more advanced topics...

## Working with Tutorial Files

To use a tutorial:

1. Click on a tutorial filename in the list
2. The tutorial will open in a new tab in the editor
3. Read the comments and explanations in the file
4. Execute the code examples to hear the results
5. Experiment by modifying the examples

## Interactive Learning

The tutorials are designed to be interactive:

- Each tutorial contains explanatory comments
- Code examples demonstrate the concepts
- You can execute any line or section with the standard execution shortcuts
- Try modifying the examples to experiment with the concepts

## Best Practices for Learning

1. **Work sequentially** - Start with tutorial 00 and progress in order
2. **Execute each example** - Don't just read; run the code to hear the results
3. **Experiment** - Modify the examples to deepen your understanding
4. **Take notes** - Create your own tabs to save interesting patterns
5. **Revisit later** - Return to advanced tutorials as your skills improve

## Creating Custom Tutorials

You can create your own tutorial files:

1. Create a new file with explanatory comments and examples
2. Save it to your sessions folder
3. Use it to document techniques you've developed
